# オブジェクト指向教材 基本ルール

このドキュメントは、Java オブジェクト指向の教材サンプルにおける**構成の基本ルール**を定めます。デザイン（色・余白・フォントなど）は `Doc/4-デザインルール.md` を参照してください。

---

## 講座の目的と受講者像

- **受講者像**： 文系など、ロジックに強くない人でも学べる講座にする。専門用語は必要に応じて補足し、段階を踏んだ説明で、誰でもついていけるようにする。
- **講座のもう一つの目的**： 本講座は**IT の講師としての心構えを学ぶ**場でもある。教材を作る・教える立場を意識した構成・表現を心がける。受講者が将来、自分で教えるときの参考になるような「なぜそう書くか」「現場ではどうか」といった視点も盛り込む。

---

## コラムの扱い

テキストや練習問題のところどころに、**コラム**として「実務やチーム開発の視点」「講師として伝えたい心構え」を入れていく。

- **コラムの内容**： 技術的なルールの「理由」や「現場での意味」を短く説明する。例：「コメントは適度に入れるべきです。実際の Java 開発では、何十人といったチームで開発します。そのため、ほかの人が見てもわかりやすいソースコードの書き方にする必要があります。」
- **入れ方**： 時折、該当するトピックの近くに「コラム」「ひとこと」などの見出しで挿入する。長くしすぎず、1テーマあたり数行〜短い段落で収める。
- **テーマの例**： コメントの重要性、命名のわかりやすさ、チーム開発での読みやすさ、講師として意識したい説明の仕方、など。

## 基本のファイル構成

各レッスン（回）の教材は、次の4種類のファイルで構成する。

| 区分 | 内容 |
|------|------|
| **【概要】** | その回の前提知識・ポイント解説・対象テキストのページ・動画一覧をまとめたファイル。学習の入口となる。 |
| **【テキスト】** | 本編の解説（復習・学習目標・導入・説明・サンプルコード・練習問題・まとめ）を収めたファイル。 |
| **【練習問題】** | テキストと連動した演習問題（基本・発展）を収めたファイル。 |
| **【確認テスト】** | 理解度を確認するためのテストを収めたファイル。 |

- **練習問題の構成**： 練習問題は**大きく3つ**作成する。それぞれ**3段階くらいで拡張**していく形にする。
  - **基本問題①** — **動物クラスを利用した問題**（例： Animal を土台に Dog / Cat などサブクラスで拡張）。
  - **基本問題②** — **社員クラスを利用した問題**（例： Employee を土台に Manager / PartTimer などサブクラスで拡張）。
  - **発展問題③** — **繰り返しや配列などを使った少し高度な問題**（例： 配列と for で複数インスタンスを扱う）。
- **問題の設定はテーブルで見やすく**： 練習問題・確認テストにおいて、問題の設定（フィールド／メソッド一覧、ステップごとの条件、仕様など）は**見やすいように &lt;table&gt;（テーブル）を使って作成**する。
- 概要ファイルの**構成とデザイン**は `Doc/4-デザインルール.md` の「概要ファイル」の項に従う。
- **すべてのHTMLファイルの1行目**： **概要・テキスト・練習問題・確認テスト**のいずれにおいても、1行目は**タグなしのプレーンテキストでタイトル**を書く。2行目以降に本文（body内のHTML、または &lt;style&gt; から始まる場合も可）を書く。
  - **概要**： 「【概要】 オブジェクト指向〇 ～テーマ～」（例： 【概要】 オブジェクト指向③ ～継承とオーバーライド～）。
  - **テキスト**： 「【テキスト】 オブジェクト指向〇 ～テーマ～」（例： 【テキスト】 オブジェクト指向② ～コンストラクタとカプセル化～）。講義用など別名が必要な場合は「（講義用）」などを付けてよい。
  - **練習問題**： 「【練習問題】 オブジェクト指向〇 ～テーマ～」（例： 【練習問題】 オブジェクト指向③ ～継承とオーバーライド～）。レッスンに紐づかない総合練習問題集の場合は「Java研修 練習問題集」などでもよい。
  - **確認テスト**： 「【確認テスト】 オブジェクト指向〇 ～テーマ～」。
- **動画のリンク**： 動画のリンク（クリックして再生するURL）の**元データ**は **`Doc/1-カリキュラムと進行.md`** に含まれる。概要の「動画」の項では、この回で視聴する主な動画について、**カリキュラム・進行からURLを取得し、概要に直接 &lt;a href="..."&gt; のアンカータグでリンクを入れておく**。リンクは `target="_blank"` と `rel="noopener noreferrer"` を付ける。
- **アップロードと参照可能範囲**： 公開アップロードするのは**各回の概要・テキスト・練習問題・確認テスト（ある場合とない場合あり）のHTMLのみ**である。`Doc/` 配下のカリキュラム・進行などのファイルは公開していないため、受講者は参照できない。そのため、アップロードするHTML（概要・テキスト・練習問題・確認テスト）の本文中に「カリキュラムと進行を参照」など、受講者が開けないファイルへの参照を書かない。
- **テキスト**は題材を**自販機で統一**する（下記「オブジェクト指向サンプルの基本構成」参照）。**練習問題**は上記「練習問題の構成」に従う。

## HTMLファイルはbody内のみ

すべてのHTMLファイル（概要・テキスト・練習問題・確認テスト）は、**&lt;body&gt; の中身だけ**を書く。

- **書かないもの**： `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` タグは含めない。
- **書くもの**： 本来 &lt;body&gt; の内側に書くコンテンツ（&lt;div&gt;、&lt;section&gt;、見出し・本文など）のみ。&lt;head&gt; にあったスタイルが必要な場合は、本文の先頭に &lt;style&gt; を置く。
- **1行目**： 上記「すべてのHTMLファイルの1行目」のとおり、概要・テキスト・練習問題・確認テストのいずれも1行目はタイトル（タグなし）。2行目以降が本文（body内のHTML、または &lt;style&gt; を含む本文）。

## オブジェクト指向サンプルの基本構成

オブジェクト指向の教材サンプルは、**混乱を防ぐため題材を自動販売機で統一**する。

| 区分 | 内容 |
|------|------|
| **テキスト** | **自動販売機を使ったたとえ** — 解説・会話でクラス・コンストラクタ・カプセル化などを自販機のたとえで説明する。 |
| **練習問題** | **基本①動物・基本②社員・発展③** — 上記「練習問題の構成」に従い、3区分でそれぞれ3段階くらいで拡張する形で練習する。 |

- **テキスト部分**: 自販機（VendMachine など）を例に、用語と動きを説明する。
- **練習問題部分**: 上記「練習問題の構成」に従い、基本問題①（動物クラス）・基本問題②（社員クラス）・発展問題③（繰り返し・配列など）の3区分で、それぞれ3段階くらいで拡張する形で演習を用意する。

## 用語の統一（継承まわり）

継承の説明では**親クラス・子クラス**という呼称は使わず、**スーパークラス・サブクラス**に統一する。

- **スーパークラス** — 継承の元になるクラス（いわゆる「親」に相当）。教材では「親クラス」「親」とは書かず「スーパークラス」と書く。
- **サブクラス** — 継承した側のクラス（いわゆる「子」に相当）。教材では「子クラス」「子」とは書かず「サブクラス」と書く。
- コード例のプレースホルダーも「親クラス名」→「スーパークラス名」、「子クラス名」→「サブクラス名」に合わせる。

## ソースコードの見やすさとコメント

教材に掲載するソースコード（テキスト・練習問題・解答例・配布用コードなど）は、**見やすく**し、**何をしているのかわかりやすいように適度にコメントを入れる**。

- **見やすさ**： インデント（字下げ）や改行を整え、ブロックの範囲が一目でわかるようにする。長い行は適宜改行する。
- **コメント**： クラスやメソッドの役割、処理の区切り、わかりにくい箇所には**適度に**コメントを入れる。
  - 自明な行にコメントを並べすぎず、「ここで何をしているか」が伝わる程度にする。
  - クラス先頭やメソッドの上に、そのクラス・メソッドの説明を1行程度入れるとよい。
- **対象**： テキスト内のサンプルコード、練習問題の解答例、配布用の Java ファイル（src_10 など）すべてに適用する。

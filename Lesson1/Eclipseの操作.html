第1回：Eclipseの操作 ～環境に慣れる～

1. この回の目標

この回を終えたとき、次のことができるようになりましょう。

  • Eclipse を起動し、ワークスペースを選べる。
  • 新しい Java プロジェクトを作成できる。
  • クラスを作成し、main メソッドを含むプログラムを実行できる。
  • 保存と自動コンパイルの関係を理解している。

2. Eclipse とは

Eclipse は、Java のプログラムを書いたり実行したりするための統合開発環境（IDE）です。エディタ・コンパイル・実行・エラー表示がひとつの画面でまとまっているため、効率よく開発できます。

Eclipse を使う前に、JDK（Java Development Kit） をインストールしておく必要があります。動画の「概要＋JDK」「Eclipse」の回で手順を確認してください。

3. ワークスペース

Eclipse を起動すると、最初にワークスペースを選ぶ画面が出ます。ワークスペースは「この Eclipse で使う作業用フォルダ」です。プロジェクトや設定はここに保存されます。

💡 ポイント

同じワークスペースを指定すれば、前回作成したプロジェクトがそのまま表示されます。別のフォルダを選ぶと、別のワークスペースとして扱われます。

4. プロジェクトの作成

Java のプログラムはプロジェクト単位で管理します。手順の目安は次のとおりです。

  1. メニューから ファイル → 新規 → Java プロジェクト（または「新規」→「Java プロジェクト」）を選ぶ。
  2. プロジェクト名を入力（例：MyFirstProject）し、完了をクリック。

プロジェクトがパッケージ・エクスプローラーに表示されれば成功です。

5. クラスの作成と実行

プログラムを書くには、プロジェクトの中にクラスを作ります。実行するには、main メソッドが必要です。

手順の目安

  1. プロジェクトを右クリック → 新規 → クラス。
  2. クラス名を入力（例：Hello）。
  3. 「public static void main(String[] args) にチェックを入れる」にチェックを入れて完了。
  4. エディタに表示された main の中に、例えば System.out.println("Hello"); と書く。
  5. メニュー 実行 → 実行、またはツールバーの緑の三角ボタン（実行）をクリック。

コンソールに Hello と表示されれば成功です。

最小限の実行可能なコード例

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}

6. 自動コンパイルと保存

Eclipse では、ファイルを保存すると自動的にコンパイル（ビルド）が行われます。そのため、実行する前に必ず保存（Ctrl+S / Mac は Command+S）しておきましょう。

文法エラーがあると、エディタの左側や「問題」ビューに赤いマークで表示されます。エラーを直してから再度保存・実行してください。

7. まとめ

  • ワークスペース： Eclipse の作業用フォルダ。同じフォルダを選べば前回の続きから。
  • プロジェクト： Java のプログラムをまとめる単位。まず「新規 Java プロジェクト」で作成。
  • クラスと main： プログラムはクラスに書く。実行するには main メソッドが必要。
  • 保存 → 自動コンパイル → 実行： 保存すると自動でビルドされる。緑の三角で実行。

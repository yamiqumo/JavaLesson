【テキスト】 AIによるコーディング体験 — 自分のLP（ランディングページ）を作ろう

単体講座 — AIコーディング体験

AIによるコーディング体験 — チャット型AI（Gemini・ChatGPTなど）を使い、自分のサイト（LP）を1から作って公開するまで（約1時間半）

1. この回で学ぶこと

この講座では、AIを活用してコーディングする体験をします。プログラミングの「書き方」を覚えるだけでなく、「何を作るか」「どの技術で作るか」「どうやって公開するか」までを、AIとの対話を交えながら進めていきます。

  • AIによるコーディングとは何か、AIの種類（チャット型・エディタ連携型・自律型）の違いを知る。
  • 今回はチャット型AI（Gemini・ChatGPTなど）を使い、コーディングのコツや注意点を体験する。
  • 無料のホスティングサービスを利用し、自分用のLP（ランディングページ）をPHPで作成して公開する。

LP（ランディングページ）とは、1枚のWebページで「商品紹介」や「自己紹介」などのテーマに絞って伝える形式のページです。まずはシンプルな1ページから作っていきましょう。

2. AIによるコーディングとは

AIによるコーディングとは、プログラミングの一部（コードのひな型・修正・説明）を、AI（人工知能）に手伝ってもらいながら進めるやり方です。自分で「何をしたいか」を言葉で伝え、AIがコード例や説明を返してくれるので、初めての言語や初めての機能でも、調べる時間を短縮できます。

注意点として、AIの返答は必ずしも正しくないことがあります。生成されたコードは動かして確認し、エラーが出たら「どの部分がおかしいか」を一緒に切り分けていく姿勢が大切です。

3. AIの種類と特徴

コーディングを支援するAIには、大きく次の3つのタイプがあります。

① Gemini・ChatGPT などのチャット型AI

  • 1問1答のやり取りが基本。会話のたびに「今のプロジェクト全体」を把握しているわけではないので、全体の整合性を取るのは自分になります。
  • その分、とっつきやすく、やり取りが優しいので、初めてAIでコーディングするときに向いています。

② Cursor

  • VS Code（統合開発環境のエディタ）をベースにしたAIエージェントです。エディタの中でコードを書きながら、時々に応じて適切なAIを使い分けられます。
  • プロジェクト単位でファイルを把握できるので、基本仕様を決めておけば、それに沿ったコーディングや修正を任せやすくなります。
  • Git（バージョン管理）やGitHubの知識があると、変更履歴を管理しつつAIと協力しやすくなります。

③ Claude Code など（自律型）

  • 「従業員の管理アプリを作って」のように大きな指示を1回出すと、設計から実装・修正まで自律的に進め、完成に近い形まで仕上げてくれるタイプです。
  • ①→②→③と進むほど、自動化される部分が増えます。今回は入門として①のチャット型AIを使い、コツや注意点を体感しながら自分のLPを作成します。

4. 今回の進め方（約1時間半）

次の順番で進めます。思い浮かばない場合は、以下の仮のターゲットを題材に、同じ手順でLPを作成していきます。

前提条件をしっかり示す

LPを作る前に、次の6点を決めておくと、AIへの依頼や技術選定がしやすくなります。

  • 誰がアクセスするのか？
  • どんなことをさせたいのか？（閲覧だけ／問い合わせ／申し込み など）
  • 無料か有料か？いくらまで出せるのか？
  • ゆくゆくはどうしたいのか？
  • どのくらいの分量か？
  • 誰が編集するのか？
  1. 4-1 無料でサーバーを立ち上げられるサイトを探す — どこにファイルを置いて公開するか決める。
  2. 4-2 ドメインの考え方を知る — インターネット上の「住所」のようなもの。無料サブドメインでもOK。
  3. 4-3 プログラミング言語の選択 — 今回はWebページ向けなので、候補を比較する。
  4. 4-4 技術の選定 — 調べた結果、PHPで進めることにし、無料ホスティングでPHPが使えるサービスを選ぶ。
  5. 4-5 コーディングをしていく — AIに相談しながら、LP用のHTML・PHPを書いていく。
  6. 4-6 デバッグ — 表示や動作を確認し、おかしなところを直す。

4-1. 無料でサーバーを立ち上げられるサイトを探す

Webサイトを公開するには、サーバー（あなたのHTMLやPHPファイルを置いて、世界中から見られるようにするコンピュータ）が必要です。自分でサーバーを買わなくても、無料のホスティングサービスを利用すれば、小規模なLPなら無料で公開できます。

ホスティング＝サーバーの一部（または1台）を貸し出して、そこにファイルを置けるようにするサービス、という意味で覚えておくとよいです。

例として、PHPが使える無料ホスティングには「infinityfree」「000webhost」「Xfreehost」などがあります。静的サイト（HTML中心）でよい場合はFirebase Hostingもよい選択肢です。検索キーワードは「無料 ホスティング PHP」や「Firebase Hosting」などで調べてみましょう。1つ選び、そこにアップロードするところまで行います。

4-2. ドメインの考え方

ドメインとは、インターネット上でWebサイトの場所を表す「名前」です。例えば www.example.com の example.com の部分がドメインです。

独自ドメイン（自分で取得した名前）は有料のことが多いですが、多くの無料ホスティングではサブドメイン（例：あなたの名前.サービス名.com）が無料で付与されます。まずはこのサブドメインで「自分のLPにアクセスできるURL」を手に入れましょう。

4-3. プログラミング言語の選択

Webページを作るには、最低限HTML（見た目と構造）が必要です。それに加えて、サーバー側で動く処理（例：日付の表示、簡単なフォーム処理）を入れたい場合は、PHP・Python・Rubyなどの言語がよく使われます。

無料ホスティングではPHPが使えるサービスが多く、情報も豊富なので、今回のLP作成ではPHPを選びます。Javaを学んだ経験があれば、「変数」「if文」「ループ」の考え方は共通しているので、PHPの書き方の違いをAIに聞きながら進めていきましょう。

4-4. 技術の選定（PHPに確定）

ここまでの流れで、無料ホスティング ＋ PHPでLPを作る方針にします。PHPはサーバー上で実行され、HTMLの中に <?php ... ?> で処理を埋め込めます。日付を表示したり、同じヘッダー・フッターを共通化したりするのに便利です。

AIに「PHPでLPのひな型を書いて」と依頼すると、HTMLとPHPが混ざったサンプルを出してくれます。それをベースに、自分用の文言やスタイルに書き換えていきましょう。

4-5. コーディングをしていく

PHP を書くエディタは VSCode のほか、Eclipse に PHP 用プラグイン（PDT：PHP Development Tools）を入れて使うこともできます。Eclipse を使い慣れている場合は「Eclipse for PHP Developers」をダウンロードするか、既存の Eclipse に PDT を追加すれば、PHP の編集・実行・デバッグが可能です。

AI（Gemini・ChatGPTなど）への質問のコツを意識しながら進めます。

  • やりたいことを具体的に書く — 「LPを作りたい」だけでなく、「トップにキャッチコピー、その下に自己紹介文、最後に連絡先」のように書くと、それに近い構成のコードを出してくれやすい。
  • 技術を指定する — 「PHPで」「HTML5で」「インラインスタイルで」などと書くと、出てくるコードが使いやすくなる。
  • 出てきたコードは必ず実行して確認する — コピペしただけでは動かない・表示が崩れることがあるので、ローカルやホスティングのプレビューで確かめる。

深められる質問・学びのある質問を心がける

AIとのやり取りでは、工夫が必要な聞き方と、深められる・学びのある聞き方があります。次の例を参考に、自分で質問文を組み立ててみましょう。

工夫が必要な例深められる・学びのある聞き方
○○のエラーを修正したい。○○のエラーを修正するにはどうしたらいいですか？また、その理由も含めて教えてほしい。
うまく動きません。○○と△△をやったのですが、このようなエラーが出ます。（エラー文や状況をそのまま書く）
何かいい方法はありませんか？WebサイトのLPを作成したいと思っています。何の言語がおすすめですか？私はJavaを基礎まで学んでいます。
Webサイトを見れるようにしたいのですがどうしたらいいですか？PHPを使った、無料でサイトを閲覧できるサービスを考えています。おすすめはありますか？

まずは「1枚のLP用のPHPファイル」を1つ作り、タイトル・見出し・短文・フッター程度の内容から始めます。慣れてきたら、PHPで日付を表示したり、セクションを増やしたりして調整していきましょう。

4-6. デバッグ

デバッグとは、プログラムの不具合（バグ）を見つけて直す作業です。LPでよくあるのは、表示が崩れる・真っ白になる・PHPのエラーで動かないといったことです。

  • ブラウザの開発者ツール（F12キーなど）で、コンソールやネットワークのエラーを確認する。
  • PHPの場合は、ホスティングのエラーログや、画面上に表示されるエラーメッセージを手がかりに、該当する行を直す。
  • AIに「このエラーメッセージが出ている。どう直す？」と貼り付けて聞くと、修正案を出してくれる。

分からないところは、エラーメッセージや「どこまで表示されているか」をそのままAIに伝えると、切り分けのヒントをもらいやすくなります。

5. やってみてどうだったか・気づきのシェア

目的：制作したLPを発表し、得たこと・わかったこと・良かった点を共有する。

やること：作ったLPを画面で見せながら短く発表する。その後、得たもの・わかったこと・良かった点を一言ずつ共有する。

本講座では、AIの種類（チャット型・Cursor・自律型）の違いを学び、無料ホスティングとPHPでLPを制作・公開するところまで体験しました。AIへの質問のコツ（具体的に書く・技術を指定する・実行して確認する）を、今後のコーディングにも活かしてみてください。

この単元で学ぶこと

「抽象クラス」と「インタフェース」— 型の決まりをプログラムで表す仕組みです。次回の多態性の土台になります。

【1】具体的な学習目標

【必須】

  • 抽象クラスの役割を説明できる。
  • abstract class と abstract メソッドの書き方を理解する。
  • インタフェースの役割を説明できる。
  • interface と implements の書き方を理解する。

【オプション】

  • 抽象クラスとインタフェースの違いを説明できる。

【2】導入：型の決まり

「この型の変数には、こういうメソッドが必ずある」という決まりを、抽象クラスやインタフェースで表します。次回の多態性で、同じ型でいろいろな自販機を扱う準備になります。

【3】抽象クラスとは

抽象クラス（abstract class）は、そのクラス単体では new でインスタンスを作れません。abstract メソッドは中身を書かず、サブクラスで必ず実装します。VendMachine を abstract にすると、共通の buy() の流れは持つが、beforeBuy() や dispense() はサブクラスで実装する、という設計になります。

【4】インタフェースとは

インタフェース（interface）は、メソッドの「名前・引数・戻り値」だけを決めた約束です。クラスは implements インタフェース名 でその約束を実装します。例：CanSell（void buy();）を定義し、Vending2Item が implements CanSell する、など。

【5】抽象クラスとインタフェースの違い

extends は1つだけ、implements は複数可能。抽象クラスは「共通の骨組み＋サブクラスで実装」、インタフェースは「メソッドの約束だけ」を表します。

【6】練習問題・【7】まとめ

テキスト本編の【6】練習問題で、抽象クラスを継承したサブクラスの作成と、インタフェースの実装を練習します。【7】まとめでは、次回の多態性につなげることを確認します。

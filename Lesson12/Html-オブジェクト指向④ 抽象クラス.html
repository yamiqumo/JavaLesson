この単元の前に：継承の復習

前回までに「継承」と「オーバーライド」を自販機で学んでいます。短くおさらいします。

復習：前回までに学んだこと

  • 継承 — extends スーパークラス名 でサブクラスがスーパークラスの機能を引き継ぐ。
  • オーバーライド — スーパークラスと同じ名前・同じ引数のメソッドをサブクラスで定義し直すこと。オーバーライドしたメソッドには public をつけておく。
  • VendMachine — 自販機の共通の流れ（buy() など）を持ち、サブクラスで差分を実装してきた。

この単元で学ぶこと

この単元では「抽象クラス」を学びます。次回インタフェース、その次に多態性の土台になります。

【1】具体的な学習目標

この単元を終えたとき、以下のことができればゴールです。

  • 抽象クラスの役割（基本とするためのクラスとしてしか使わず、サブクラスに実装を任せる）を説明できる。
  • abstract class と abstract メソッド の書き方を理解する。
  • 「1つでも抽象メソッドがあるクラスは抽象クラス」であることを理解する。

【2】導入：基本とするクラスに具体的なメソッドは要らない

カバ先生：「サブクラスで書き換えるのなら、そもそもスーパークラスに具体的に書く必要はないよね。なので、基本とするためのクラスとしてしか使わないなら、スーパークラスのメソッドを抽象メソッドにします。」

ユウタ：「1つでも抽象メソッドがあるクラスが、抽象クラスなんですね。」

【3】抽象クラス：設計図だけのクラス

抽象クラスは、そのクラス単体では new でインスタンスを作れないクラスです。クラス名の前に abstract を付けます。

  • abstract class クラス名 — 抽象クラス。サブクラスで継承して使うことを前提にする。
  • abstract メソッド — 中身（本体）を書かず、サブクラスで「必ず実装する」ことを強制する。
  • 抽象クラスを継承したサブクラスは、すべての abstract メソッドを実装しないとコンパイルエラーになる。

サンプル：抽象クラス VendMachine

自販機の「買う」流れは共通にしたいが、中身（在庫の持ち方・チェックの仕方）はサブクラスで違う、という設計です。

// 抽象クラス：new できない。サブクラスで実装する前提。
public abstract class VendMachine {

    public void buy() {
        if (beforeBuy() == true) {
            System.out.println("1分ほどお待ちください");
            dispense();
        }
    }

    // サブクラスで必ず実装する（中身は書かない）
    protected abstract boolean beforeBuy();
    protected abstract void dispense();
}

【4】練習問題

【問題】抽象クラスを継承する
VendMachine が抽象クラスで protected abstract boolean beforeBuy(); と protected abstract void dispense(); を持っているとき、SimpleVend というサブクラスを作り、beforeBuy() では常に true を返し、dispense() では「商品を出しました」と表示するように実装してください。
▶ ヒント

public class SimpleVend extends VendMachine とし、@Override で beforeBuy() と dispense() を実装します。オーバーライドしたメソッドには public をつけておきましょう。

【問題】Vending2Items クラスを自分で書いてみる
VendMachine を継承し、2種類の商品を扱う Vending2Items クラスを自分で書いてみましょう。
仕様：フィールドに drinkName1・drinkName2（String）、stock1・stock2・selected（int）を持ち、buy() では selected の値（1 または 2）に応じて対応する在庫を減らし「選択したアイテムを購入した」と分かる表示をすること。
▶ ヒント

extends VendMachine とし、beforeBuy() では selected が 1 または 2 かつ対応する在庫が 1 以上かどうかを返す。dispense() では selected に応じて stock1 または stock2 を減らし、どちらの商品を出したか表示する。

▶ 解答例

【解説・確認用】selected が 1 のときは商品1、2 のときは商品2を出します。抽象メソッド beforeBuy() と dispense() を実装します。

public class Vending2Items extends VendMachine {
    private String drinkName1, drinkName2;
    private int stock1, stock2, selected;

    public Vending2Items(String n1, String n2, int s1, int s2, int sel) {
        drinkName1 = n1; drinkName2 = n2;
        stock1 = s1; stock2 = s2; selected = sel;
    }

    @Override
    protected boolean beforeBuy() {
        if (selected == 1 && stock1 >= 1) return true;
        if (selected == 2 && stock2 >= 1) return true;
        System.out.println("在庫がありません");
        return false;
    }

    @Override
    protected void dispense() {
        if (selected == 1) {
            stock1--;
            System.out.println(drinkName1 + "を購入しました");
        } else {
            stock2--;
            System.out.println(drinkName2 + "を購入しました");
        }
    }
}

【5】まとめ

  • 抽象クラス — abstract class で宣言。そのクラスでは new できない。1つでも抽象メソッドがあるクラスが抽象クラス。
  • 抽象メソッドはサブクラスで必ず実装する。オーバーライドしたメソッドには public をつけておく。

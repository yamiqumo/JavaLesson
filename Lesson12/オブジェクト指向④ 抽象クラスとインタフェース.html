抽象クラスとインタフェース（約2時間）

この単元の前に：アクセス修飾子と継承の復習

前回は「メンバの可視性（public / private / protected）」を学びました。その前には「継承」と「オーバーライド」を自販機で学んでいます。短くおさらいします。

復習：前回までに学んだこと

  • アクセス修飾子 — public（どこからでも）、private（クラス内だけ）、protected（サブクラスからも）で、メンバの見え方を決める。
  • 継承 — extends スーパークラス名 でサブクラスがスーパークラスの機能を引き継ぐ。
  • オーバーライド — スーパークラスと同じ名前・同じ引数のメソッドをサブクラスで定義し直すこと。
  • VendMachine — 自販機の共通の流れ（buy() など）を持ち、サブクラス（Vending2Item、VendingHotNoodle）で差分を実装してきた。

この単元で学ぶことのイメージ

この単元では「抽象クラス」と「インタフェース」を学びます。どちらも「型の決まり」をプログラムで表す仕組みです。次回の多態性（同じ型でいろいろな自販機を扱う）の土台になります。

【1】具体的な学習目標

この単元を終えたとき、以下のことができればゴールです。

【必須】

  • 抽象クラスの役割（インスタンスを直接作らず、サブクラスに実装を任せる設計図）を説明できる。
  • abstract class と abstract メソッド の書き方を理解する。
  • インタフェースの役割（メソッドの「形」だけを約束する）を説明できる。
  • interface と implements の書き方を理解する。

【オプション】

  • 抽象クラスとインタフェースの違い（継承は1つ、インタフェースは複数実装可能など）を説明できる。
  • インタフェースを実装するクラスのメソッドには、自動的に public が付くことを理解する。

【3】抽象クラス：設計図だけのクラス

抽象クラスは、そのクラス単体では new でインスタンスを作れないクラスです。クラス名の前に abstract を付けます。

  • abstract class クラス名 — 抽象クラス。サブクラスで継承して使うことを前提にする。
  • abstract メソッド — 中身（本体）を書かず、サブクラスで「必ず実装する」ことを強制する。
  • 抽象クラスを継承したサブクラスは、すべての abstract メソッドを実装しないとコンパイルエラーになる。

サンプル：抽象クラス VendMachine

自販機の「買う」流れは共通にしたいが、中身（在庫の持ち方・チェックの仕方）はサブクラスで違う、という設計です。

// 抽象クラス：new できない。サブクラスで実装する前提。
public abstract class VendMachine {

    public void buy() {
        if (beforeBuy() == true) {
            System.out.println("1分ほどお待ちください");
            hotWaterServe();
            dispense();
        }
    }

    // サブクラスで必ず実装する（中身は書かない）
    protected abstract boolean beforeBuy();
    protected abstract void dispense();

    protected void hotWaterServe() {
        // デフォルトは何もしない。必要なサブクラスだけオーバーライド
    }
}

このように abstract class VendMachine と書くと、new VendMachine() はできません。代わりに Vending2Item や VendingHotNoodle のようなサブクラスで beforeBuy() と dispense() を実装して使います。

【4】インタフェース：メソッドの約束

インタフェースは、メソッドの「名前・引数・戻り値」だけを決めた約束です。中身は書きません。クラスは implements インタフェース名 で、その約束を実装します。

  • interface 名前 — メソッドの形だけを定義。本体は ; で終える。
  • class クラス名 implements インタフェース名 — そのクラスがインタフェースで決めたメソッドをすべて実装する。
  • インタフェースで宣言したメソッドをクラスで実装するとき、実装側のメソッドには自動的に public が付く（前回学んだアクセス修飾子の「どこからでも見える」になる）。
  • 1つのクラスは、継承（extends）は1つだけだが、インタフェース（implements）は複数実装できる。

サンプル：「売れる」インタフェース

「buy() というメソッドを持つ」という約束をインタフェースで表します。

// インタフェース：メソッドの形だけを決める
public interface CanSell {
    void buy();  // 中身は書かず ; で終える
}

// クラスが「CanSell を実装する」と宣言
public class Vending2Item extends VendMachine implements CanSell {
    // ... 既に buy() を持っているので、CanSell の約束を満たしている。実装メソッドは自動的に public
}

このように implements CanSell と書くと、「このクラスは buy() を持っています」と型として示せます。次回の多態性で、CanSell 型の変数にいろいろな自販機を入れて、同じ buy() を呼ぶ、という使い方ができます。

【5】抽象クラスとインタフェースの違い（整理）

項目 抽象クラス（abstract class） インタフェース（interface）
継承・実装の数 extends は1つだけ implements は複数可能
メソッドの中身 普通のメソッドも持てる。abstract メソッドは中身なし メソッドは形だけ（中身なし）
用途のイメージ 共通の骨組み＋「ここをサブクラスで実装」 「このメソッドを持つ」という約束だけ

ナナコ：「抽象クラスは設計図、インタフェースは約束。次は同じ型でいろいろな自販機を扱う多態性なんですね。」

【6】練習問題（2問）

【問題0】抽象クラスを継承する
VendMachine が抽象クラスで protected abstract boolean beforeBuy(); と protected abstract void dispense(); を持っているとき、SimpleVend というサブクラスを作り、beforeBuy() では常に true を返し、dispense() では「商品を出しました」と表示するように実装してください。
ヒント

public class SimpleVend extends VendMachine とし、@Override で beforeBuy() と dispense() を実装します。コンストラクタで super(name, stock) を呼ぶ必要がある場合は、引数を受け取って渡してください。

解答例 
public class SimpleVend extends VendMachine {
    public SimpleVend(String name, int stock) {
        super(name, stock);
    }
    @Override
    protected boolean beforeBuy() { return true; }
    @Override
    protected void dispense() {
        System.out.println("商品を出しました");
    }
}


【問題1】インタフェースを実装する
CanSell インタフェース（void buy(); のみ）を定義し、Vending2Item が implements CanSell するようにしてください。Vending2Item はすでに buy() を持っているので、インタフェースを追加するだけでかまいません。
ヒント

interface は public interface CanSell { void buy(); } のように書きます。クラス側は public class Vending2Item extends VendMachine implements CanSell と、implements CanSell を追加します。

解答例 
// CanSell.java
public interface CanSell {
    void buy();
}

// Vending2Item のクラス宣言の部分のみ
public class Vending2Item extends VendMachine implements CanSell {
    // 既存のまま。buy() があるので CanSell の約束を満たしている
}

【8】まとめのおさらい

  • 抽象クラス — abstract class で宣言。そのクラスでは new できない。abstract メソッドはサブクラスで必ず実装する。
  • インタフェース — interface でメソッドの形だけを定義。implements でクラスがその約束を実装する。実装メソッドは自動的に public。
  • 抽象クラスもインタフェースも「型の決まり」を表し、次回の多態性の土台になる。

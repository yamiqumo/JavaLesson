この単元で学ぶこと

「抽象クラス」と「インタフェース」— 型の決まりをプログラムで表す仕組みです。次回の多態性の土台になります。

【1】具体的な学習目標

【必須】

  • 抽象クラスの役割を説明できる。
  • abstract class と abstract メソッドの書き方を理解する。
  • インタフェースの役割を説明できる。
  • interface と implements の書き方を理解する。

【オプション】

  • 抽象クラスとインタフェースの違いを説明できる。インタフェースの実装メソッドには自動的に public が付くことを理解する。

【2】導入・【3】抽象クラス・【4】インタフェース・【5】違いの整理

テキスト本編の【2】〜【5】で、型の決まりを表す「抽象クラス」と「インタフェース」、および「実装メソッドは public が付く」ことを学びます。

【6】練習問題・【7】再確認・【8】まとめ

1. この回の目標

この回を終えたとき、次のことができるようになりましょう。ロジックに慣れていなくても、順を追って進めれば大丈夫です。

  • ソースコードの基本構造（クラス・メソッド）とコメントの書き方を理解している。
  • System.out.print と println の違いを説明できる。
  • 整数・小数・文字列のリテラルを書いて表示できる。
  • 型（int・double・String）を指定して変数を宣言し、値を代入・表示できる。

2. ソースコードの構造とコメント

Java のプログラムはクラスの中にメソッドを書きます。実行の入口は main メソッドです。字下げ（インデント）や改行で構造を整えると読みやすくなります。

コメントは実行されません。// から行末までが1行コメント、/* ... */ で囲んだ部分が複数行コメントです。

// サンプル用クラス。実行の入口は main メソッド
public class Sample {
    public static void main(String[] args) {
        /* 複数行のコメントも書ける */
        System.out.println("Hello");  // コンソールに表示
    }
}

コラム：コメントは適度に入れよう

実際の Java 開発では、何十人といったチームで開発することがあります。そのため、ほかの人が見てもわかりやすいソースコードを書く必要があります。コメントは「ここで何をしているか」が伝わる程度に、適度に入れましょう。入れすぎるとかえって読みづらくなることもあります。

3. print と println

System.out.print(〇〇) は「〇〇を表示し、改行しない」、System.out.println(〇〇) は「〇〇を表示し、そのあと改行する」です。

// print は改行しない。println は表示後に改行する
System.out.print("A");
System.out.print("B");
System.out.println("C");   // ここで改行。コンソールには "ABC" と1行で出る

4. リテラル

リテラルは、プログラムに直接書く値です。主な種類は次のとおりです。

種類 型の例 リテラルの例
整数int0, 100, -5
小数double3.14, -0.5
文字列String"Hello", "こんにちは"（ダブルクォートで囲む）

これらを println に渡すと、そのままコンソールに表示されます。

5. 変数と型

変数は「データを入れる箱」です。Java では型を指定して宣言し、その型に合った値だけを代入します。

宣言の形： 型 変数名; または 型 変数名 = 初期値;

// 変数の宣言と代入（型 変数名 = 値;）
int age = 20;
double height = 1.65;
String name = "太郎";
// 変数の値をコンソールに表示
System.out.println(name);
System.out.println(age);
System.out.println(height);

💡 ポイント

変数名は識別子のルールに従います（英数字・アンダースコアなど。数字で始めない）。型と代入する値の種類が一致している必要があります。

6. まとめ

  • 構造： クラス → メソッド。実行は main から。コメントは // と /* */。
  • print / println： println は表示後に改行。
  • リテラル： 整数・小数・文字列（ダブルクォート）をそのまま書ける。
  • 変数と型： 型 変数名 = 値; で宣言と代入。型に合った値だけ代入する。

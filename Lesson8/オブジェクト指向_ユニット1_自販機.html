ユニット①：オブジェクト指向って何？（約2時間）

このユニットの前に：前回までに学んだこと

前回（Lesson7）は制御構文（if 文・for 文・while 文など）を学びました。その前にはメソッドの作成（引数・戻り値）にも触れています。このユニットでは、自販機を例に「クラスとインスタンス」「フィールドとメソッド」を学びます。

  • 制御構文 — if で条件分岐、for ・ while で繰り返し。条件式は true / false で判定します。
  • メソッド — 引数を受け取り、戻り値を返す「処理のまとまり」。main もメソッドの一つです。

このユニットでは「クラスとインスタンス」「フィールドとメソッド」「定義する側と呼び出す側」を自販機で学び、最後にカプセル化のイメージにも軽く触れます。

  • カプセル化のイメージを説明できる。

【3】自販機のサンプルプログラム

注意：話をシンプルにするため、飲み物は一旦「水」だけ、処理も「メッセージを出すだけ」にします。

カバ先生：「役割をはっきりさせるために、『自販機を作る側のファイル（定義）』と『自販機を置いて動かす側のファイル（実行）』の2つに分けてプログラムを書くよ。」

ファイル1：【定義する側】VendingMachine.java（自販機の設計図）

// 自販機のルールを決める「メーカー」の視点
public class VendingMachine {

    // フィールド（自販機が持っているデータ）
    String drinkName = "水";

    // メソッド（自販機ができる動き）
    public void buy() {
        // 動作をシンプルに
        System.out.println("水を購入しました");
    }
}

ファイル2：【呼び出す側】Main.java（自販機を使うオーナーの視点）

// 自販機を設置して動かす「オーナー」の視点
public class Main {

    public static void main(String[] args) {
        // 設計図（クラス）から本物の自販機を作る（インスタンス化）
        VendingMachine myMachine = new VendingMachine();
        // 作った自販機のボタンを押してみる（呼び出し）
        myMachine.buy();
    }
}

【5】練習問題（2問）

【問題1】クラスとインスタンスの判別
次のペアについて、どちらがクラスで、どちらがインスタンスでしょうか？
・「車の設計図」と「今、駐車場に止まっている車」
ヒント

「設計図」はそれだけでは動かない・使えないものです。「今、駐車場に止まっている車」は実際に存在する1台の車です。

解答

クラス：車の設計図 … 型・ルールを決めたもの。
インスタンス：今、駐車場に止まっている車 … 設計図から作られた実体（1台の車）。



【問題2】視点を変えて考えてみよう！
あるモノをクラス（型）またはインスタンス（実体）と見なしたとき、それと対になる例を考えてみましょう。
1. 「お弁当」をインスタンス（実体）としたとき、そのクラス（型）にあたるものは何でしょうか？
2. 「大理石のハンコ」をクラス（型）としたとき、そのインスタンス（実体）にあたるものは何でしょうか？
ヒント

1. お弁当は「誰かが作った1個の実物」です。その「作り方・種類を決めているもの」が型です。
2. ハンコそのものは「同じ形で何度も押せる型」です。その型を使って紙に押した「結果」が実体です。

解答

1. クラス（型）：「お弁当のレシピ」「〇〇弁当のメニュー」など、お弁当の種類・作り方を決めているもの。
2. インスタンス（実体）：「ハンコで押した印鑑」「紙に付いたハンコの跡」など、ハンコを使ってできた1つ分の結果。

【6】ハンズオン：自販機をパワーアップさせよう！

カバ先生：「今はボタンを押すと決まった文字が出るだけだけど、次はフィールド（データ）をメソッド（動き）の中で活用する練習をしてみよう。本来、フィールドはメソッドで利用するための情報を入れておく場所なんだよ。」

① フィールドを使って名前を表示しよう

カバ先生：「まずは、フィールドの drinkName を使って購入した商品を表示させてごらん。」

【問題3】 VendingMachine.java の buy() メソッドを修正し、drinkName の値を使って「〇〇を購入しました」と表示するようにしてください。
ヒント

System.out.println の中で、文字列 "を購入しました" の前に drinkName を + でつなぎます。例：変数 + "文字列"。

解答例 
// VendingMachine.java の buy() の中身を修正
public void buy() {
    // フィールドの変数を使って表示する
    System.out.println(drinkName + "を購入しました");
}

ユウタ：「あ！これなら将来 drinkName を『お茶』に書き換えたとき、メソッドの中身をいじらなくても表示が勝手に変わるんだね。便利！」

② 在庫数を管理しよう

カバ先生：「次は、自販機に在庫数（stock）を持たせてみよう。初期値を10とし、購入するたびに1減らすプログラムを追加するんだ。」

【問題4】 VendingMachine クラスに在庫用のフィールド stock（int）を追加し、初期値を10にしてください。また、buy() の中で在庫を1減らし、残り在庫を表示するようにしてください。
ヒント

フィールドに int stock = 10; を追加します。buy() 内で stock--; で1減らし、System.out.println("残りの在庫は" + stock + "個です"); で表示します。

解答例 
public class VendingMachine {

    String drinkName = "水";
    int stock = 10;  // 追加：在庫の初期値は10

    public void buy() {
        System.out.println(drinkName + "を購入しました");
        stock--;  // 追加：在庫を1減らす
        System.out.println("残りの在庫は" + stock + "個です");
    }
}

ナナコ：「メソッドを実行するたびにフィールドの stock が減っていくのね。自販機が自分の状態を『覚えている』感じがするわ。」

【発展】③ 在庫ゼロの時は「売り切れ」にしよう

カバ先生：「今のままだと、在庫が0になってもマイナスになって売り続けちゃうよね。在庫が0の時は販売できないようにしてみよう。」

【問題5】 buy() メソッドを修正し、在庫が0より大きいときだけ「購入しました」と在庫を減らし、0以下のときは「売り切れです」と表示するようにしてください。
ヒント

if (stock > 0) { ... } else { ... } で分岐します。stock > 0 のときだけ購入処理（表示・stock--）を実行し、そうでなければ「売り切れです」を表示します。

解答例 
public void buy() {
    if (stock > 0) {
        System.out.println(drinkName + "を購入しました");
        stock--;
        System.out.println("残りの在庫は" + stock + "個です");
    } else {
        // 在庫が0以下の時
        System.out.println("売り切れです");
    }
}

ユウタ：「これで本物の自販機っぽくなった！『在庫というデータ』を見て、『売るか売らないかという動き』を変えているんだね。」

【8】まとめのおさらい

  • クラスは設計図（レシピ）、インスタンスは実体（料理）。
  • フィールドは「モノの状態」を保存し、メソッドはその「状態」を利用して動く。
  • プログラムは「ルールを決める側（定義側）」と「それを使う側（呼び出す側）」で分担する。

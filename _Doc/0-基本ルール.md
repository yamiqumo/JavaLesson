# オブジェクト指向教材 基本ルール

このドキュメントは、Java オブジェクト指向の教材サンプルにおける**構成の基本ルール**を定めます。デザイン（色・余白・フォントなど）は `_Doc/4-デザインルール.md` を参照してください。

---

## 講座の目的と受講者像

- **受講者像**： 文系など、ロジックに強くない人でも学べる講座にする。専門用語は必要に応じて補足し、段階を踏んだ説明で、誰でもついていけるようにする。
- **講座のもう一つの目的**： 本講座は**IT の講師としての心構えを学ぶ**場でもある。教材を作る・教える立場を意識した構成・表現を心がける。受講者が将来、自分で教えるときの参考になるような「なぜそう書くか」「現場ではどうか」といった視点も盛り込む。

---

## コラムの扱い

テキストや練習問題のところどころに、**コラム**として「実務やチーム開発の視点」「講師として伝えたい心構え」を入れていく。

- **コラムの内容**： 技術的なルールの「理由」や「現場での意味」を短く説明する。例：「コメントは適度に入れるべきです。実際の Java 開発では、何十人といったチームで開発します。そのため、ほかの人が見てもわかりやすいソースコードの書き方にする必要があります。」
- **入れ方**： 時折、該当するトピックの近くに「コラム」「ひとこと」などの見出しで挿入する。長くしすぎず、1テーマあたり数行〜短い段落で収める。
- **テーマの例**： コメントの重要性、命名のわかりやすさ、チーム開発での読みやすさ、講師として意識したい説明の仕方、など。

## 基本のファイル構成

各レッスン（回）の教材は、次の4種類のファイルで構成する。

| 区分 | 内容 |
|------|------|
| **【概要】** | その回の前提知識・ポイント解説・対象テキストのページ・動画一覧をまとめたファイル。学習の入口となる。 |
| **【テキスト】** | 本編の解説（復習・学習目標・導入・説明・サンプルコード・練習問題・まとめ）を収めたファイル。 |
| **【練習問題】** | テキストと連動した演習問題（基本・発展）を収めたファイル。 |
| **【確認テスト】** | 理解度を確認するためのテストを収めたファイル。 |

- **教材ファイルのファイル名**： **`.cursorrules` に規定あり**（Exe-練習問題○○ / Html-××② ▲▲ / Base-××② ▲▲ / pdf-××② ▲▲ / Test-確認テスト○○ / Desc-概要○○。複数ある場合は最後に①などを付与）。

- **練習問題の構成**： 練習問題は**大きく3つ**作成する。それぞれ**3段階くらいで拡張**していく形にする。
  - **基本問題①** — **動物クラスを利用した問題**（例： Animal を土台に Dog / Cat などサブクラスで拡張）。
  - **基本問題②** — **社員クラスを利用した問題**（例： Employee を土台に Manager / PartTimer などサブクラスで拡張）。
  - **発展問題③** — **繰り返しや配列などを使った少し高度な問題**（例： 配列と for で複数インスタンスを扱う）。
- **問題の設定はテーブルで見やすく**： 練習問題・確認テストにおいて、問題の設定（フィールド／メソッド一覧、ステップごとの条件、仕様など）は**見やすいように &lt;table&gt;（テーブル）を使って作成**する。
- 概要ファイルの**構成とデザイン**は `_Doc/4-デザインルール.md` の「概要ファイル」の項に従う。
- **すべてのHTMLファイルの1行目・body内のみ**： **`.cursorrules` に規定あり**（1行目＝タグなしタイトル、出力は body 内のみ）。種別ごとのタイトル例： 概要「【概要】 オブジェクト指向〇 ～テーマ～」、テキスト「【テキスト】 ～」、練習問題「【練習問題】 ～」または「Java研修 練習問題集」、確認テスト「【確認テスト】 ～」。
- **動画のリンク**： 動画のリンク（クリックして再生するURL）の**元データ**は **`_Doc/1-カリキュラムと進行.md`** に含まれる。概要の「動画」の項では、この回で視聴する主な動画について、**カリキュラム・進行からURLを取得し、概要に直接 &lt;a href="..."&gt; のアンカータグでリンクを入れておく**。リンクは `target="_blank"` と `rel="noopener noreferrer"` を付ける。
- **アップロードと参照可能範囲**： 公開アップロードするのは**各回の概要・テキスト・練習問題・確認テスト（ある場合とない場合あり）のHTMLのみ**である。`_Doc/` 配下のカリキュラム・進行などのファイルは公開していないため、受講者は参照できない。そのため、アップロードするHTML（概要・テキスト・練習問題・確認テスト）の本文中に「カリキュラムと進行を参照」など、受講者が開けないファイルへの参照を書かない。
- **テキスト**の題材は **`.cursorrules` のとおり自販機で統一**する。**練習問題**は上記「練習問題の構成」に従う。

## HTMLファイル（body内のみ・1行目・スタイル）

**`.cursorrules` に規定あり**（body 内のみ、1行目はタグなしタイトル、スタイルはインラインで &lt;style&gt; は使わない）。ここでは省略する。

## オブジェクト指向サンプルの基本構成

題材・テキスト/練習の役割は **`.cursorrules` と同一**。テキスト＝自販機（VendMachine）のたとえ、練習問題＝基本①動物・基本②社員・発展③でそれぞれ3段階程度で拡張。

---

## 講義用テキストの基本クラスとハンズオン

- **前回までの基本クラスを土台にする**  
  この単元で追加する内容をいちいち詳しく記述しなくてよい。**前回までの基本クラス**を掲載し、それを土台に受講者が**ハンズオンで挑戦**するスタイルとする。
- **基本クラスの掲載**  
  講義用テキストでは、基本クラス（土台コード）を折りたたみ可能な textarea でドラッグコピーできる形で掲載する。基本クラスの読み込み・配布の扱いは `.cursorrules` および `_Doc/4-デザインルール.md` の「講義用テキストの基本クラス・コピー用エリア」に従う。

---

## 用語の統一（継承まわり）

**`.cursorrules` に規定あり**（「親/子クラス」は使わず「スーパークラス/サブクラス」に統一）。プレースホルダーも「スーパークラス名」「サブクラス名」に合わせる。

---

## ソースコードの見やすさとコメント

**`.cursorrules` の Java コーディング規約**に従う（フィールド・コンストラクタ・getter/setter の直前コメントなど）。加えて： インデント・改行を整え、自明な行にコメントを並べすぎず、クラス/メソッドの説明は1行程度。対象はテキスト・練習問題・配布用コードすべて。

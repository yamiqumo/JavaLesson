## 練習問題 基本仕様

### 1. 目的
セクション・問題・ヒント・解答を一貫した形式で提示するための共通フォーマット。**デザイン（色・ヒント/解答のHTMLテンプレート）は `_Doc/4-デザインルール.md` に一元化する。**

---

### 2. 全体構成

- **タイトル**: ファイル先頭に演習全体のタイトルを1つ（例：`# Java研修 練習問題集`）。
- **セクション**: 難易度・テーマごとに `##` で区切る（例：基本問題① 動物 / 基本問題② 社員 / 発展問題③）。**発展問題③も動物・社員のクラスを用いる。**
- **問題ブロック**: 各セクション配下に、下記3〜5の仕様に従う問題ブロックを並べる。

**練習問題の題材（必須）**

- **練習問題で使う題材は動物（animal）と社員（employee）のみ**とする。
- **自販機（VendingMachine）はテキスト本編のみのサンプル**とし、練習問題の配布コード・問題文では使わない。
- 基本①＝動物クラス（例：Cryable / Dog / Cat）、基本②＝社員クラス（例：Payable / Manager / PartTimer）、発展③＝配列・繰り返し等（動物または社員のクラスで出題）。

---

### 2.1 配布コード（練習問題）

- **方針**: 練習問題では、**基本のクラスをコピペできるように別ファイルで配布**する想定とする。
- **掲載位置**: **配布コードは練習問題ファイルの最初**（タイトル・説明の直後）にまとめて入れておく。受講者がそのままコピペして使えるようにする。
- **手動配布**: 講師が手動で別ファイル（.java やテキスト）として渡す場合は、その旨を問題文に記載し、割り振り（どのファイルをいつ渡すか）を明記する。

---

### 2.2 練習問題の流れ（アクセス修飾子・カプセル化の回）

アクセス修飾子やカプセル化を扱う回では、次の流れで問題を構成する。

1. **① public を private に修正し、アクセスできないことを確認する**  
   フィールドを `public` から `private` に変え、Main などから直接参照するとコンパイルエラーになることを確認する。
2. **② private にしたので、ゲッター・セッターを設定する**  
   外部から読む・書くために getter / setter を追加する。
3. **③ ゲッター・セッターを介して、Main からアクセスできることを確認する**  
   Main で getter / setter を使って値の取得・変更ができることを確認する。
4. **④ セッターにフィルタリングを設定する**  
   不正な値を防ぐため、setter 内で範囲チェックなどを行う。例：年齢は 0～150 歳のみ受け付け、それ以外は変更させない。

---

### 3. 問題ヘッダと基本情報

1. **問題番号と概要**
   - 1行で問題番号と短い説明を記載する。
   - フォーマット:  
     `**【問題n】** 説明文`
   - 例:  
     `**【問題1】** 1から10までを順番に表示してください。`
   - `n` はファイル全体で通し番号とする。

2. **メソッド仕様（必要な場合のみ）**
   - 問題で特定のメソッド実装を求める場合、問題文直後にメタ情報を列挙する。
   - 例:
     - `**メソッド名**：greet`
     - `**引数**：String message`
     - `**戻り値**：void`
   - 複数引数がある場合はカンマ区切りとする。  
     例: `**引数**：String str1, String str2`

3. **区切り線**
   - 問題文とヒント・解答のブロックを視覚的に分ける。**スタイルは `_Doc/4-デザインルール.md` の「④ 練習問題の区切り線」に従う**（`margin: 10px 0`）。

4. **各問題ブロックの構成**（順序を守る）
   - **【問題】** — 問題文を太字で記載。
   - **ヒント** — `<details><summary>ヒント</summary>...</details>`（summary に ▶ は書かず、ブラウザ標準マーカーのみ）。
   - **解答** — `<details><summary>解答例</summary>...</details>` のなかに、解説（任意）・pre（カラー）・**ドラッグ用** `<div>` + `<textarea>` を入れる。**HTML の詳細は `_Doc/4-デザインルール.md` の「⑤ 練習問題の解答ブロック」に従う。**

---

### 4. ヒント・解答

- ヒントは `<details><summary>ヒント</summary>...</details>`。解答は `<details><summary>解答例</summary>...</details>` で、その中に解説・pre・ドラッグ用 textarea を必ず含める。スタイル・テンプレートは **`_Doc/4-デザインルール.md` の「③ インタラクティブ・トグル」「⑤ 練習問題の解答ブロック」** を参照。

---

### 4.1 解答・サンプルコードの記述ルール

解答およびテキスト内のサンプルコードは **`.cursorrules` の「3. Javaコーディング規約」** に従う。戻り値は変数に代入してから return、文字列比較は .equals()、型名を明示（var は使わない）、**インデントはタブ**（コピペ時の崩れ防止）、**コンストラクタ・メソッドは本体に処理がある場合は1行にまとめない（中身が空のコンストラクタは `public Foo() {}` のように1行でよい）**、**わかりやすいコメントを適宜入れる**（基本クラス設計は `_Doc/0-基本ルール.md` の「自販機の基本クラス設計」を参照）。

---

### 4.2 サンプルコードのコメント（学習の妨げにならない程度）

教材のサンプルコード（配布コード・解答例・テキスト本編）には、**学習の妨げにならない程度**で、次のようなコメントを入れる。どこに何があるかがひと目で分かるようにし、抽象クラス・継承・インタフェース・インスタンス化などの用語がコード上で対応できるようにする。

**用語とコメントの対応（目安）**

| 対象 | コメント例 |
|------|------------|
| **クラス定義** | `// クラス定義` または `// ○○クラス（××を継承）` |
| **抽象クラス** | `// 抽象クラス` または `// スーパークラス（抽象クラス）` |
| **継承** | `// ○○クラスを継承` の直上またはクラス宣言の行 |
| **インタフェース実装** | `// ○○インタフェースを実装` の直上またはクラス宣言の行 |
| **インタフェース** | `// インタフェース（××という約束）` |
| **フィールド** | `// フィールド` をまとまりの上に。必要なら各フィールドに短く |
| **コンストラクタ** | `// コンストラクタ` または `// コンストラクタ1` / `// コンストラクタ2` |
| **メソッド** | `// メソッド（××処理）` または `// ○○の約束を実装` |
| **ゲッター・セッター** | `// ゲッター` / `// セッター` |
| **main 内のインスタンス化** | `// インスタンス化（○○型の変数に △△を代入）` など |

区切りが必要なときは `// ----------` のような区切り線コメントを使ってよい。**入れすぎず、構造が読める程度**を心がける。

---

### 5. 名前の付け方（教材用）

- **クラス名・メソッド名**：できるだけ破綻しない程度に簡単にする。
  - 例：自販機クラスは `VendingMachine` で統一する。
- **インスタンス変数（参照変数）**：簡易でよい。
  - 例：`VendingMachine vm = new VendingMachine();`（`vm` のように短くしてよい）
  - 複数ある場合は `vm`、`vm2` などでよい。
- **フィールド名**：文脈が分かる範囲で簡易でよい。
  - 例：`drinkName` → `name`、getter は `getDrinkName()` → `getName()` など。

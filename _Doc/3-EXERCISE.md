## 練習問題ファイル 基本仕様（EXERCISE_BASE.md）

### 1. 目的
- **目的**: Java などのプログラミング演習問題を、セクション・問題・ヒント・解答付きで一貫した形式で提示するための共通フォーマットを定義する。

---

### 2. ファイル全体構成

- **タイトル**:
  - ファイル先頭に演習全体のタイトルを1つ記載する。
  - 例: `# Java研修 練習問題集` または `## Java研修 練習問題集`
- **セクション**:
  - 制御構文や難易度ごとに `##` 見出しで区切る。
  - 例:
    - `## 基本問題（for文）`
    - `## 基本問題（while文）`
    - `## 基本問題（if文）`
    - `## 基本問題（switch文）`
    - `## 基本問題（break文）`
    - `## 基本問題（continue文）`
    - `## 発展問題（for文）`
    - `## 発展問題（while文）`
    - `## 発展問題（if文）`
    - `## 発展問題（switch文）`
    - `## 複合・総合問題`
- **問題ブロック**:
  - 各セクション配下に、以下3〜6の仕様に従う「問題ブロック」を複数並べる。

---

### 2.1 配布コード（練習問題）

- **方針**: 練習問題では、**基本のクラスをコピペできるように別ファイルで配布**する想定とする。
- **掲載位置**: **配布コードは練習問題ファイルの最初**（タイトル・説明の直後）にまとめて入れておく。受講者がそのままコピペして使えるようにする。
- **手動配布**: 講師が手動で別ファイル（.java やテキスト）として渡す場合は、その旨を問題文に記載し、割り振り（どのファイルをいつ渡すか）を明記する。

---

### 2.2 練習問題の流れ（アクセス修飾子・カプセル化の回）

アクセス修飾子やカプセル化を扱う回では、次の流れで問題を構成する。

1. **① public を private に修正し、アクセスできないことを確認する**  
   フィールドを `public` から `private` に変え、Main などから直接参照するとコンパイルエラーになることを確認する。
2. **② private にしたので、ゲッター・セッターを設定する**  
   外部から読む・書くために getter / setter を追加する。
3. **③ ゲッター・セッターを介して、Main からアクセスできることを確認する**  
   Main で getter / setter を使って値の取得・変更ができることを確認する。
4. **④ セッターにフィルタリングを設定する**  
   不正な値を防ぐため、setter 内で範囲チェックなどを行う。例：年齢は 0～150 歳のみ受け付け、それ以外は変更させない。

---

### 3. 問題ヘッダと基本情報

1. **問題番号と概要**
   - 1行で問題番号と短い説明を記載する。
   - フォーマット:  
     `**【問題n】** 説明文`
   - 例:  
     `**【問題1】** 1から10までを順番に表示してください。`
   - `n` はファイル全体で通し番号とする。

2. **メソッド仕様（必要な場合のみ）**
   - 問題で特定のメソッド実装を求める場合、問題文直後にメタ情報を列挙する。
   - 例:
     - `**メソッド名**：greet`
     - `**引数**：String message`
     - `**戻り値**：void`
   - 複数引数がある場合はカンマ区切りとする。  
     例: `**引数**：String str1, String str2`

3. **区切り線**
   - 問題文とヒント・解答のブロックを視覚的に分ける。
   - 推奨: `<div style="border-bottom: 1px solid #ccc; margin-bottom: 10px;"></div>`

4. **各問題ブロックの構成**（順序を守る）
   - **【問題】** — 問題文を太字で記載。
   - **ヒント** — `<details><summary>▶ ヒント</summary>...</details>`
   - **解答** — `<details style="margin-top:10px;"><summary>▶ 解答</summary>...</details>` のなかに、解説（任意）・カラー表示用 `<pre>` ・**ドラッグ貼り付け用**の `<div>` + `<textarea>`（プレーンテキストの Java コード、user-select:all）を入れる。詳細な HTML テンプレートは **`_Doc/4-デザインルール.md` の「⑤ 練習問題の解答ブロック」** に従う。

---

### 4. ヒントブロック

- ヒントは 0 個以上。折り畳みは `<details>` / `<summary>` を使用。summary は **▶ ヒント**。
- 解答側の summary は **▶ 解答**。解答内に「解説・確認用」の p、pre（カラー）、ドラッグ用 textarea を含める（4-デザインルール参照）。

---

### 4.1 解答・サンプルコードの記述ルール

解答およびテキスト内のサンプルコードは **`.cursorrules` の「3. Javaコーディング規約」** に従う。戻り値は変数に代入してから return、文字列比較は .equals()、型名を明示（var は使わない）、メソッド上部および main 内の呼び出し箇所に解説コメントを入れる。

---

### 5. 名前の付け方（教材用）

- **クラス名・メソッド名**：できるだけ破綻しない程度に簡単にする。
  - 例：`VendingMachine` → `VendMachine`
- **インスタンス変数（参照変数）**：簡易でよい。
  - 例：`VendMachine vm = new VendMachine();`（`vm` のように短くしてよい）
  - 複数ある場合は `vm`、`vm2` などでよい。
- **フィールド名**：文脈が分かる範囲で簡易でよい。
  - 例：`drinkName` → `name`、getter は `getDrinkName()` → `getName()` など。

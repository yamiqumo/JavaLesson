## 2. 相談内容で決まった方針（要約）

- **static とアクセス修飾子** … 講義で扱う。**実施順は「アクセス修飾子を先に、抽象・インタフェースを後に」が推奨**（インタフェースの「実装クラスには public が付く」話をするときに、先に public の意味を学んでいると理解しやすい）。static（例：自販機の台数など）、アクセス修飾子（public / private / protected）の両方を追加する。
- **継承とオーバーライド** … Lesson10 で実施済み。このあとの回で復習として触れるのはあり。本編としてもう一度やる必要はない。
- **コレクションは扱わない** … 講義では **例外のみ** とする。
- **DB** … 講義の **最後に 1 日**、DB の基礎を挿入する。
- **演習** … DB のあと、**簡単なコンソールアプリ**を **2 週間** で作成してもらう。
  - **基本**（全員）… オブジェクト指向を使わず、ロジックを組む力を試す。
  - **基本2** … これまでのオブジェクト指向の流れを踏襲して実装。
  - **発展** … 機能拡張して実用的に。
  - **自由課題** … 独自の機能を実装。
- 演習中はコンテンツを自分で見て学び、わからなければ復習して進める。**完成したらアップロードしてもらい、通知する**。

## 3. 今後のスケジュール予定（講義 → DB → 演習）

| 順番 | 内容 | 備考 |

| **済** | Lesson8 … クラスとインスタンス | 自販機で設計図と実物のイメージ |
| **済** | Lesson9 … コンストラクタとカプセル化 | 初期化・private・getter/setter |
| **済** | Lesson10 … 継承とオーバーライド | extends・オーバーライド・super。以降は復習で触れる形で可 |
| **予定** | **Lesson11 … アクセス修飾子（メンバの可視性）** | public / private / protected の整理。教材あり。**先に実施し、その次に Lesson12（抽象・インタフェース）。** |
| **予定** | **Lesson12 … 抽象クラスとインタフェース** | abstract・interface・implements。実装メソッドは自動的に public。教材あり。 |
| **予定** | **static**（1回） | static メンバ（自販機の台数など）。アクセス修飾子と同日 or 別回で可。教材は未作成。 |
| **予定** | **多態性(1)** | 同じ型で異なる動き。アップキャスト・オーバーライドの活用 |
| **予定** | **多態性(2)・応用** | ポリモーフィズムの実践・インタフェースの効果 |
| **予定** | **例外** | try-catch・例外の扱い。**コレクションは扱わない**。 |
| **予定** | **予備 / 総復習**（必要に応じて） | 質疑・補習など |
| **予定** | **DB の基礎**（1日） | 講義の締めとして 1 日で実施。 |
| **予定** | **演習**（2週間） | コンソールアプリ。基本 → 基本2 → 発展 → 自由課題。完成したらアップロード・通知。 |

※ 日付・回数は `_Doc/1-カリキュラムと進行.md` や `_Doc/1-2カリキュラム修正案.md` と合わせて調整してください。  
※ **明日すでに「抽象クラスとインタフェース」で決まっている場合**は、今回に限りそのまま実施してよく、**次回以降のカリキュラム**から「先にアクセス修飾子、次に抽象・インタフェース」の順を採用するとよい。

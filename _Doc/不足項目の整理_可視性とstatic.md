## ① メンバの可視性（public / private / protected）

### 現状

- **Lesson9（オブジェクト指向② コンストラクタとカプセル化）**
  - 【6】カプセル化で **private** と **public**（getter・setter）は説明している。
  - 「クラスの内側からだけ」「外からは見えません」という表現はあるが、**「可視性」という用語で public / private / protected をまとめて扱う節はない**。
- **Lesson10・Lesson11**
  - コード例で **protected** が出てくるが、「可視性」や「protected の意味」の解説はしていない。

### 入れる場所

- **Lesson9** に、次のどちらかで追加するのが自然です。
  1. **【6】カプセル化** の「private と getter・setter」の**前**に、見出し「メンバの可視性」を 1 つ作り、**public（どこからでも）・private（クラス内だけ）・protected（サブクラスからも）** を短くまとめる。
  2. または **【6】の冒頭**で「カプセル化の前に、メンバの可視性（public / private / protected）を押さえましょう」とし、表や箇条書きで 3 つを整理してから、private と getter・setter の説明に入る。

これで、Lesson10 以降で `protected` が出てきても「可視性の一種」として復習しやすくなります。

## ② static メンバの意味（自販機の台数を数える）

### カリキュラム上の位置

- **`_Doc/1-カリキュラムと進行.md`** の **No.10**（2/6 金）：
  - **Chapter 7 オブジェクトの仕組み (2)**（7.3–7.4）
  - 狙い：**static と参照の深い理解**
  - 動画：103. クラスメンバ、104. メンバ混在、105. 参照とは、106. 参照型と null、108. イミュータブル

- **`_Doc/2-テキスト作成ガイダンス.md`** の復習一覧にも、
  - No.10：**Chapter 7 (2)** — **static（クラスメンバ）**、インスタンスとクラスの違い、参照型、null、イミュータブル  
  と明記されています。

### 教材の対応状況

- 現在の流れは次のとおりです。
  - **Lesson8** … クラスとオブジェクト（No.8 相当）
  - **Lesson9** … コンストラクタとカプセル化（No.9 相当）
  - **Lesson10** … 継承とオーバーライド（No.11 相当）
- このため、**No.10（static と参照）に相当する「1 回」が、教材には存在していません**。  
  「自販機の台数を数える」のような **static フィールドの例**も、現状のどの Lesson にもありません。

### 入れる場所

- **Lesson9 の次・Lesson10（継承）の前**に、**1 回ぶんの Lesson を新設**する形が、カリキュラムと一致します。
  - 例：**「オブジェクトの仕組み(2) static・参照」** のようなタイトルで 1 回。
  - 内容の例：
    - **static メンバの意味**（クラスに 1 つだけあるフィールド／メソッド、インスタンスの数と無関係）
    - **自販機の台数を数える** — 例：`private static int machineCount;` を用意し、コンストラクタで `machineCount++;` する。全台数は「クラスに 1 つ」なので static が適している、という説明。
    - （時間があれば）参照型・null のイメージ、イミュータブル（String）に軽く触れる。

- 回数のずれ方：
  - 現状：Lesson10 ＝ 継承（カリキュラム No.11）。
  - 対応案：**「static と参照」の回を 1 回挿入**し、その後を次のようにする。
    - 新設回（static・参照） → 継承（現 Lesson10）→ 抽象クラス・インタフェース（現 Lesson11）→ 多態性(1) → 多態性(2) → コレクション・例外 → 予備/総復習  
  こうすると、**あと 6 回**で No.10〜16 までカバーできます。

## 方針の整理（2025/2 時点）

- **static とアクセス修飾子** … いったんここで入れる方針でよい。static（自販機の台数など）と、アクセス修飾子（public / private / protected）の両方を追加する。
- **継承とオーバーライド** … 前回（Lesson10）で実施済み。このあとの回で復習として触れるのはあり。本編としてもう一度やる必要はない。
- **コレクションは入れない** … 講義パートでは**例外のみ**扱う。コレクション＋DB の組み合わせはやらない。
- **DB** … 講義の**最後に 1 日**、DB の基礎を挿入する。
- **演習** … DB のあと、**簡単なコンソールアプリ**を**2 週間**で作成してもらう。基本／基本2／発展／自由課題の 4 段階。完成したらアップロードしてもらい、通知する。詳細は `_Doc/演習とDBの流れ.md` を参照。

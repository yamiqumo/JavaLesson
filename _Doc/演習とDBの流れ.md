# 演習と DB の流れ

講義の終盤から演習までの流れと、演習の構成・進め方をまとめています。

---

## 1. 講義の終盤の流れ

- **Lesson10** … 継承とオーバーライド（済）
- このあと … static・アクセス修飾子、抽象クラス・インタフェース、多態性、**例外** など
- **コレクションは扱わない**。講義では **例外のみ** とする。
- **最後に 1 日** … **DB の基礎** を挿入する（講義の締めとして 1 日で実施）。

---

## 2. 演習の位置づけ

- DB のあと、**演習**として **簡単なアプリ** を作成してもらう。
- **期間** … **2 週間**
- **形式** … 基本的に **コンソールアプリ**（コンソールアプリを作成してもらう）。

---

## 3. 演習の 4 段階（基本・基本2・発展・自由課題）

| 段階 | 内容 | ねらい |
|------|------|--------|
| **基本** | 全員が作成する。**オブジェクト指向を使わず**に実装する。 | **ロジックを組む力**を主に試す。条件分岐・繰り返し・メソッドなどで処理を書けるかを確認する。 |
| **基本2** | これまで学んだ**オブジェクト指向の流れを踏襲**して実装する。 | クラス・コンストラクタ・カプセル化・継承などを活かした設計・実装ができるかを確認する。 |
| **発展** | 機能拡張をして**実用的なもの**にしていく。 | 基本2 のアプリに機能を足し、使いやすい・実務に近い形にしていく。 |
| **自由課題** | **独自の機能**を実装する。 | 自分で要件を考え、設計・実装まで行う。 |

- **基本**は必ず全員に作成してもらう。
- 基本2 → 発展 → 自由課題 は、時間と意欲に応じて進めてもらう想定。

---

## 4. 学習の進め方（2 週間中）

- **コンテンツは自分で見て学んでいく**（教材・動画などを自習）。
- **わからなければ復習**してから再度取り組む流れにする。
- 講師・サポートは質問対応やレビューでフォローする。

---

## 5. 完成後の扱い（アップロードと通知）

- **完成したら**、成果物を**アップロード**してもらう。
- アップロードを検知して**通知**する仕組みを用意する（例：リポジトリへの push 通知、提出フォルダの変更通知など）。
- 通知先・方法は運用に合わせて設定する。

---

## 6. まとめ（流れの一覧）

1. 講義 … 例外まで（コレクションは扱わない）。
2. **DB の基礎** … 最後に 1 日挿入。
3. **演習** … 2 週間。
   - 基本（OO なし・ロジック） → 基本2（OO 踏襲） → 発展（機能拡張） → 自由課題（独自機能）。
   - 自学・復習しながら進め、完成したらアップロード → 通知。
